## 🎯 Objective

Configure the **CyberArk Privileged Access Management (PAM)** vault and onboard privileged accounts securely.

---

# 📋 Procedure

## 🏗️ Step 1: CyberArk Component Overview

> **Purpose:** Understand core CyberArk architecture and components used in Privileged Access Management.

### Core Components

* 🔐 **Digital Vault (PAS):**

  * Encrypted storage for privileged credentials

* 🌐 **PVWA (Password Vault Web Access):**

  * Web interface for requesting and accessing credentials

* 🔄 **CPM (Central Policy Manager):**

  * Automatically rotates passwords based on defined policies

* 🎥 **PSM (Privileged Session Manager):**

  * Session proxy for RDP/SSH
  * Records privileged sessions for auditing

* 🔌 **PSMP (SSH Proxy):**

  * Secure SSH gateway for privileged session routing

---

## 🔑 Step 2: Log in to PVWA & Create Safe

> **Purpose:** Create a secure container (Safe) for privileged credentials.

### Access PVWA

Navigate to:

```text id="cyb1"
https:///PasswordVault
```

> Log in using PAM administrator credentials with **MFA enabled**.

---

### Create a New Safe

Navigate to:

```text id="cyb2"
Policies → Safes → Add Safe
```

### Safe Configuration

* **Safe Name:** `WinAdmins_Prod`
* Assign:

  * CPM
  * Owner: PAM Admin
  * Members: IT Operations Team

---

### Safe Retention Settings

* Version retention period: **e.g., 90 days**
* Required number of days: **0 (no expiry)**


### Verification

---

## 🔐 Step 4: Set Password Policy

> **Purpose:** Enforce strong password and access governance policies for privileged accounts.

### Platform Configuration

Navigate to:

```text id="cyb4"
Administration → Platform Management → Select Platform → Edit
```

---

### Password Policy Settings

Configure:

* **Password Complexity:** 20 characters
* **Password Expiry:**

  * 24 hours (break-glass accounts), OR
  * 30 days (standard accounts)
* **Automatic Password Management:** Enabled

---

### Access Control Settings

* Enable **Dual Control (4-eyes approval)** for production admin accounts
* Enable **Just-in-Time (JIT) access** for elevated privileges

---

# 📊 Best Practice Tips

### 📌 Vault Everything

* All privileged accounts must be stored in the vault.
* Goal: **Zero Standing Privilege Access (ZSP)**.

---

### 📌 Tiered Safe Segmentation

Organize safes by environment tier:

* 🔴 Tier 0: Domain Controllers
* 🟠 Tier 1: Servers
* 🟡 Tier 2: Workstations

---

### 📌 Enforce Dual Control for Tier 0

* Tier 0 accounts require **two-person approval** before password checkout.

---

### 📌 Regular Access Reviews

* Review Safe membership **quarterly**
* Remove inactive or stale users immediately

---

### 📌 Monitor Credential Usage

Detect anomalies such as:

* Credentials checked out but not used
* Suspicious session initiation patterns

> ⚠️ These may indicate credential theft or misuse
