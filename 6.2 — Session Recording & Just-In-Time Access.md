## 🎯 Objective

Enable **Privileged Session Manager (PSM)** session proxy and configure **Just-In-Time (JIT)** privilege elevation for secure privileged access control.

---

# 📋 Procedure

## 🎥 Step 1: Configure PSM for RDP Session Recording

> **Purpose:** Enforce session monitoring, isolation, and recording for all privileged RDP sessions.

### Navigate to Platform Settings

Go to:

```text id="psm1"
PVWA → Administration → Platform Management → WindowsServerLocalAdministrator
```

---

### Enable Session Monitoring Controls

Configure the following settings:

* **RequirePrivilegedSessionMonitoringAndIsolation = Yes**
* **RecordAndSaveSessionActivity = Yes**

---

### PSM Architecture Flow

* User connects through:

  * 👤 User → 🔐 PSM → 🎯 Target System
* Direct RDP to target systems is **not allowed**

---

### Session Recording Behavior

* Sessions are recorded in **video format (DVR)**
* Recordings are securely stored in the **CyberArk Vault**
* Recording cannot be disabled by the user

---

## 💻 Step 2: Initiate PSM-Protected RDP Session

> **Purpose:** Access privileged systems securely through the PSM gateway without exposing credentials.

### Start Session

Navigate to:

```text id="psm2"
PVWA → Accounts → Select 10.1.1.50 Administrator → Connect → Connect with PSM
```

---

### Session Execution Flow

* Browser launches RDP session through PSM
* Credentials are never exposed to the end user
* Session recording starts automatically at session initiation

---

### Security Controls During Session

* All keystrokes are logged
* All activity is searchable within audit logs
* User cannot disable recording or bypass controls

---

### Post-Session Access

After session completion:

* Recording becomes available in:

  * **PVWA → Reports → Privileged Session Monitoring**

---

## ⏱️ Step 3: Configure Just-In-Time (JIT) Access

> **Purpose:** Eliminate standing privileges by granting temporary access only when required.

### Navigate to JIT Configuration

Go to:

```text id="psm3"
PVWA → Administration → Access Control → Configure JIT Settings
```

---

### Configure JIT Policies

Set the following:

* **Maximum Access Duration:** 4 hours
* **Require Business Justification:** Yes

---

### JIT Access Workflow

1. User requests elevated access
2. Manager reviews and approves request in PVWA
3. Access is granted for a limited time window
4. After expiration:

   * Account is automatically deprovisioned OR
   * Password is automatically rotated

---

## 📊 Step 4: Review Session Recordings for Audit

> **Purpose:** Monitor and analyze privileged session activity for compliance and threat detection.

### Access Session Reports

Navigate to:

```text id="psm4"
PVWA → Reports → Privileged Session Reports
```

---

### Session Review Process

* Search sessions by:

  * User
  * Date
  * Target system

* Actions:

  * Play session recording
  * Review user activity
  * Analyze commands executed

---

### Export & Integration

* Export session logs as text files
* Forward logs to SIEM for correlation

---

### Suspicious Activity Indicators

Flag sessions containing:

* Unusual or unauthorized commands
* Large file transfers
* Registry modifications

---

### SIEM Integration

* Forward CyberArk Vault audit logs via **syslog**
* Enable centralized monitoring and correlation

---

# 📊 Best Practice Tips

### 📌 Enforce Session Recording Everywhere

* Enable session recording for all privileged users:

  * IT staff
  * Vendors
  * Contractors

---

### 📌 Secure Storage of Recordings

* Store session recordings only in the **CyberArk Vault**
* Do NOT store recordings in file shares or unsecured storage

---

### 📌 Real-Time Command Monitoring

Configure alerts for high-risk commands such as:

* `net user /add`
* `reg import`
* `bcdedit`

---

### 📌 Strict Vendor Access Controls

Vendor access must always include:

* Just-In-Time (JIT) access
* Dual approval
* Session recording

---

### 📌 Conduct Regular Audits

* Perform random reviews of privileged sessions
* Acts as a strong deterrent against misuse
