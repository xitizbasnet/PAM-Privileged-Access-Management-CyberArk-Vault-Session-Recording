## 🎯 Objective

Configure **Endpoint Privilege Manager (EPM)** and enforce endpoint **least-privilege access controls** using CyberArk.

---

# 📋 Procedure

## 🖥️ Step 1: Endpoint Privilege Manager (EPM) Overview

> **Purpose:** Enforce least-privilege principles on endpoints by removing local admin rights and controlling application elevation.

### Core Concepts

* EPM enforces **least privilege** on endpoints:

  * Users operate as **standard users** by default
* Applications requiring administrative privileges can be elevated via policy
* Uses a **whitelist-based security model**
* Eliminates the need for local admin rights on workstations
* Prevents malware execution and persistence that relies on admin privileges

---

## ⚙️ Step 2: Create Application Policy

> **Purpose:** Define controlled elevation rules for approved applications.

### Navigate to EPM Console

Go to:

```text id="epm1"
EPM Console → Policies → Add Policy
```

### Policy Configuration

* **Policy Type:** Elevation

---

### Define Target Application

Applications can be identified using:

* File hash
* Publisher certificate
* File path

---

### Configure Action

* **Action:** Run Elevated

  * Grants administrative token only for the specified application

---

### User Justification Requirement

* Enable:

  * Require user justification
* Purpose:

  * Creates audit trail for elevation requests

---

### Policy Scope

Apply policy to:

* All Workstations group, OR
* Specific Organizational Unit (OU)

---

## 🚫 Step 3: Block Unknown Applications

> **Purpose:** Prevent execution of untrusted or malicious applications.

### Create Restriction Policy

Navigate to:

```text id="epm2"
EPM → Policies → Add Restriction Policy
```

---

### Policy Configuration

* **Action:** Block and Alert
* Target:

  * Unsigned executables
  * Unknown applications

---

### Allowed Exceptions

Define approved software catalog using:

* File hash
* Publisher-signed certificates

---

### Monitoring & Review

Monitor via:

```text id="epm3"
EPM Dashboard → Blocked Events
```

Review frequency:

* Weekly analysis of blocked events

> Purpose: Identify legitimate applications that should be added to the allowlist

---

### Security Impact

* Blocks most commodity malware
* Prevents execution of unsigned binaries
* Reduces risk from auto-run and script-based attacks

---

# 📊 Best Practice Tips

### 📌 Remove Local Admin Rights

* Remove local admin privileges from all workstations
* This single control mitigates a large percentage of critical vulnerabilities

> Reference: Microsoft security guidance

---

### 📌 Use Application Discovery Mode First

* Run discovery mode for **2–4 weeks**
* Collect baseline application behavior before enforcing restrictions

---

### 📌 Prefer Certificate-Based Control

* Certificate-based elevation is more resilient than hash-based rules
* Survives application updates without breaking policies

---

### 📌 Audit All Elevation Events

* Monitor repeated elevation requests for the same application
* Frequent elevation may indicate need for permanent policy adjustment

---

### 📌 Combine EPM with MFA

* Require Multi-Factor Authentication (MFA) for elevation requests
* Prevents attackers from abusing privilege elevation workflows
